# Incidents

Post-incident notes and root cause analyses from Claude sessions.

## When to Create

- CI pipeline breaks that required multiple sessions to fix
- Production issues discovered during development
- Data quality problems found in bronze/silver/gold layers
- Security issues identified during code review

Incident notes will be created in Phase 2, either manually or by a future
hook that detects incident-like patterns (e.g., multiple fix: commits in one session).
Feeds back into `global-lessons.md` for cross-project knowledge.
